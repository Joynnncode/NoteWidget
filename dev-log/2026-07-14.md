## Issues encountered / decisions made
- While investigating the Enter-key bug, found that both of SwiftUI's built-in multi-line input options are broken in this project — `TextField(axis: .vertical)` selects all the text on Enter, and `TextEditor` has the text-rendering-clipping bug on this machine (hit already back in stage 1). So a hand-rolled `MultilineTextField` was built by wrapping `NSTextView` via `NSViewRepresentable`; any new input box going forward should use this instead of either built-in option
- The "paste image" button (reading directly from the clipboard on click) simply wouldn't respond to clicks no matter what was tried (different icon, layout safeguards); ended up dropping this feature and keeping drag-and-drop only (confirmed sufficient by the user) — root cause never found, noting it here in case clipboard support is revisited later
- The image and input box were originally two separate boxes, which looked odd visually (the image floated outside, the input box narrowed and stopped growing with the window) — fixed by putting the image and text into the same rounded container
- When first translating the GitHub repo to English, scope was confirmed with the user as: just add an English README, keep the detailed dev docs in `docs/`/`dev-log/` in Chinese (internal working notes, not worth fully translating). This was revisited later the same day — see below.
- The user then asked for no Chinese to appear anywhere on GitHub at all, superseding the earlier decision above. Confirmed with the user: translate everything (`docs/`, `dev-log/`, `CLAUDE.md`); leave existing commit history untouched rather than force-pushing a rewrite. Two Chinese string literals remain in `AppDelegate.swift` (the legacy tag names `"法语学习"`/`"常规"`) since they're functional — they must match the exact old data values for the one-time rename migration to work, not display text.
