## 遇到的问题 / 决定
- 排查 Enter 键 bug 时发现：SwiftUI 自带的两种多行输入方案在这个项目里都有毛病——`TextField(axis: .vertical)` 按 Enter 会全选文字，`TextEditor` 在这台机器上文字渲染会被裁切（阶段1就踩过）。所以干脆自己用 `NSViewRepresentable` 包 `NSTextView` 写了个 `MultilineTextField`，以后新增输入框都用这个，不要再用前两种
- "贴图"按钮（点击直接读剪贴板）死活点不动，换图标、加布局保护都没解决，最后干脆去掉这个功能，只留拖拽（用户确认拖拽够用了）——原因没有查清楚，记一笔以防以后又想加类似功能
- 图片和输入框一开始做成两个独立的框，视觉上很奇怪（图片飘在外面、输入框变窄不再跟着拉伸），改成图片和文字放进同一个圆角容器里解决
- GitHub 仓库改成英文这件事，跟用户确认了范围：只加英文 README，`docs/`、`dev-log/` 里的详细开发文档保持中文（这些是内部工作记录，翻译全部性价比不高）
