## Issues encountered / decisions made
- The project folder was empty beforehand, so this was built from scratch with no legacy code to work around
- Staying on top and visible in full screen was expected to be the hard technical part; planned to use `NSPanel` + `.fullScreenAuxiliary`, treated as the key verification item for stage 4. This was tried briefly during stage 1 already, but clicking/rendering was unstable, so it was rolled back to a normal window to get basic functionality working first, leaving full screen to be tackled properly in stage 4
- A few gotchas came up during debugging, recorded in `docs/04-execution-plan.md`'s stage 1 "gotchas" section and in `docs/02-technical-design.md`:
  1. `TextEditor` has a rendering-clipping bug on this machine, switched to `TextField(axis: .vertical)`
  2. `NSApp.delegate` reads as `nil`, switched to our own `AppDelegate.shared` singleton
  3. The floating widget's and list window's `NSWindow.level` must match, otherwise the new window gets blocked and won't open
  4. Debugging kept getting derailed by leftover background processes producing contradictory feedback — learned to `pkill -9 -f NoteWidget` before every test to ensure a clean environment
  5. Switched to packaging as a `.app` and launching with `open` for everyday testing, closer to real-world usage
  6. Adding the `colorHex` field to `Tag` with a default only in the `init` parameter (not at the property declaration) caused automatic migration of old data to fail and crash the app on launch; fixed by giving the default directly at the property declaration. Since the database only had test data at the time, `~/Library/Application Support/default.store*` was simply deleted and recreated — need to be more careful once there's real data

- While testing the global shortcut, found that the floating widget doesn't show once a video enters true system full screen — this is an expected, known limitation (to be solved in stage 4), not a new bug
- Gotchas hit during stage 4 (full details in `docs/04-execution-plan.md`'s stage 4 "gotchas" section):
  1. Adding `.fullScreenAuxiliary` to the floating widget alone isn't enough — the list window also needs the same `.nonactivatingPanel` treatment to display over full screen
  2. `NSApp.activate(ignoringOtherApps:)` must not be called when showing/toggling windows, since it kicks the system out of full screen back to the regular desktop
  3. Windows need to be created up front, before entering full screen — creating a new window only after the other app is already full screen means the system won't put it into that full-screen Space
