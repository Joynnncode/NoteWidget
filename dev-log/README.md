# 开发日志说明

每天开发结束（或告一段落）时，在这个文件夹里新建一个当天的日志文件，命名格式：`YYYY-MM-DD.md`。

## 日志格式

```markdown
# YYYY-MM-DD 开发日志

## 今日完成
- ...

## 遇到的问题 / 决定
- ...

## 待办事项（下次继续）
- ...
```

## 用途

- 让每天的开发进度有据可查，方便回顾"上次做到哪了"。
- 待办事项要具体，方便下次直接接着做，不用重新想。
- 对应的整体阶段进度请看 [../docs/04-执行步骤.md](../docs/04-执行步骤.md)（这里是"每天做了什么"，那里是"整个项目走到哪个阶段"）。
