# 技术方案

对应需求见 [01-需求文档.md](01-需求文档.md)。

## 技术栈

- **Swift + SwiftUI**：界面
- **AppKit（NSPanel）**：悬浮窗口的置顶/全屏可见控制（SwiftUI 原生窗口能力不够，需要手动接管）
- **SwiftData**：本地数据存储（macOS 14+ 苹果官方方案，无需额外数据库软件）
- **KeyboardShortcuts**（第三方开源库，通过 Swift Package Manager 引入）：全局快捷键

## 工程结构

使用 **Swift Package Manager**（`Package.swift`）来组织工程，而不是手写 `.xcodeproj` 工程文件。原因：`.xcodeproj` 是复杂的内部格式，手工生成容易出细微错误导致 Xcode 打不开；而 `Package.swift` 是一个简单的文本文件，可以准确、可靠地生成，并且**可以直接用 Xcode 打开、运行、调试**（Xcode 原生支持 Swift Package 项目，效果和 `.xcodeproj` 完全一样），后续如果想转成正式的 `.xcodeproj` 也很容易。

```
Package.swift                  — 工程配置（依赖、目标）
NoteWidget.app/                — 用于日常测试的 .app 包装（见下方"开发环境注意事项"）
Sources/NoteWidget/
  NoteWidgetApp.swift          — App 入口（只有一个空的 Settings Scene，实际窗口都由 AppDelegate 手动管理）
  AppDelegate.swift            — 管理悬浮窗口 + 列表窗口 + 全局快捷键，对外暴露 AppDelegate.shared 单例
  AppModelContainer.swift      — 共享的 SwiftData ModelContainer
  FloatingPanel.swift          — `FloatingPanel`（不抢焦点的 NSPanel 子类）+ `ClickThroughHostingView`（第一次点击就生效的 NSHostingView 子类）
  Models/
    Note.swift                 — 笔记数据模型（文字、时间、标签，之后加图片）
    Tag.swift                  — 标签数据模型（名称 + 自定义颜色 colorHex）
  Views/
    FloatingWidgetView.swift   — 悬浮框界面：标签选择菜单 + 图片/文字输入区 + 打开列表按钮 + 隐藏按钮 + 保存按钮
    NoteListView.swift         — 主窗口：标签筛选条 + 倒序笔记列表（卡片样式）+ 每条笔记可删除
    NoteEditView.swift         — 点击笔记后弹出的编辑窗口，可查看/移除图片
    ThumbnailImageView.swift   — 缩略图组件，点击可放大查看完整图片
    TagFilterBar.swift         — 标签筛选条（每个标签用自己的自定义颜色）
    TagManagementView.swift    — 标签管理弹窗：新增/删除标签、改标签颜色
  ColorHex.swift               — Color 与十六进制字符串互转的小工具（存储/读取标签颜色用）
  NSImagePNG.swift             — NSImage 转 PNG Data 的小工具（存储图片用）
  GlobalHotKey.swift           — 定义全局快捷键名字（`.toggleWidget`，默认 ⌃⌥N）
  Theme.swift                  — 统一的 `Color.brandPink` 品牌粉色常量
```

没有用 `Assets.xcassets` 存主题色/图标——纯 SPM 可执行文件里管理资源目录比较繁琐，直接用 `Theme.swift` 里的代码常量定义颜色，效果一样、更可靠。App 图标目前也还没做，暂不影响功能使用。

## 关键技术点

### 1. 悬浮置顶 + 全屏可见（已完成）

悬浮框和列表窗口现在都是 `FloatingPanel`（`NSPanel` 子类）：

- `styleMask` 都包含 `.nonactivatingPanel`（悬浮框额外是 borderless + `.resizable`；列表窗口保留 `.titled/.closable/.miniaturizable/.resizable`，两者不冲突，可以同时要"不抢焦点"和"有标题栏/关闭按钮/可缩放"）。
- `isFloatingPanel = true`，`level = .floating`。
- `collectionBehavior = [.canJoinAllSpaces, .fullScreenAuxiliary, .stationary, .ignoresCycle]`（列表窗口不需要 `.ignoresCycle`）——`.fullScreenAuxiliary` 是让窗口能出现在别的 App 全屏画面之上的关键设置。
- `FloatingPanel` 覆写了 `canBecomeKey`（返回 `true`，让无标题栏的悬浮框也能接收键盘输入）和 `canBecomeMain`（返回 `false`，不抢主窗口地位）。
- 悬浮框的内容用 `ClickThroughHostingView`（`NSHostingView` 子类，覆写 `acceptsFirstMouse` 返回 `true`）而不是 `NSHostingController`，这样"不抢焦点"面板第一次点击就能直接生效，不用先点一下激活、第二下才真正触发按钮。列表窗口因为需要 `.sheet`（笔记编辑弹窗）正常工作，还是用 `NSHostingController`。
- **两个窗口都在 App 启动时就一次性建好**（悬浮框直接显示，列表窗口建好后立刻 `orderOut` 隐藏），而不是"点了才临时创建"——实测如果等到别的 App 已经进入全屏之后才新建窗口，系统不会把新窗口纳入那个全屏 Space。
- **显示/隐藏窗口时不要调用 `NSApp.activate(ignoringOtherApps:)`**（只在 App 刚启动时用一次），否则会在全屏播放时把系统切回普通桌面。日常显示/隐藏只用 `makeKeyAndOrderFront(nil)` / `orderOut(nil)`。
- 这一块调试花了很久才摸清楚，完整踩坑记录见 [04-执行步骤.md](04-执行步骤.md) 阶段1、阶段4部分。

### 2. 关键组件之间如何互相调用

**不要用 `NSApp.delegate as? AppDelegate` 来拿 AppDelegate 实例** —— 实测这个环境下 `NSApp.delegate` 会读出 `nil`（原因未完全查清，怀疑跟只用 `Settings` Scene、没有 `WindowGroup` 有关），导致按钮点击"看起来没反应"。正确做法：`AppDelegate` 提供 `static var shared: AppDelegate!`，在 `applicationDidFinishLaunching` 里赋值，SwiftUI 视图里统一用 `AppDelegate.shared.xxx()` 调用。

### 3. 文本输入框

**不要用 `TextEditor`** —— 实测在这个系统/工具链组合下有渲染裁切 bug（文字只显示下半部分）。统一用 `TextField("提示文字", text: $x, axis: .vertical).textFieldStyle(.plain).lineLimit(n...m)` 代替，视觉效果类似（自动换行、随内容变高）且没有裁切问题。

### 4. 全局快捷键（已完成）

用 `KeyboardShortcuts` 开源库（`Sources/NoteWidget/GlobalHotKey.swift` 定义了 `.toggleWidget` 这个快捷键名字），默认绑定 **⌃⌥N**（避开 Safari 等常见系统/App 快捷键冲突）。`AppDelegate.toggleWidget()` 负责显示/隐藏悬浮框。以后想改快捷键，可以加一个 `KeyboardShortcuts.Recorder` 到设置界面，让用户自己录制，不需要改代码逻辑。
- 已知限制：这个快捷键能在悬浮框"以普通身份"存在时正常显示/隐藏，但如果视频进入了系统级全屏（独立 Space），悬浮框本身还不会跟过去——这是阶段4要解决的问题，跟快捷键本身无关。

### 5. 数据模型（SwiftData）

- `Note`：`id`、`text`、`createdAt`（创建时自动写入，不可修改）、`tag`（关联到 Tag，可为空）、`imageData: Data?`（可选字段，存图片的 PNG 数据）
- `Tag`：`id`、`name`（如 "French Learning"、"General"）、`colorHex`（用户自定义颜色，存成十六进制字符串，配合 `ColorHex.swift` 转换）
- App 首次启动自动预置 "French Learning"（粉色）、"General"（灰色）两个标签，用户可以在"Manage Tags"里改颜色、增删。早期版本默认标签是中文名字，`AppDelegate.renameLegacyChineseTagsIfNeeded()` 做了一次性改名迁移。
- **重要**：给 SwiftData 模型加新字段时要在属性声明处直接写默认值（`var colorHex: String = "#FF6B9D"`），不能只在 `init` 参数里给默认值，否则旧数据自动迁移会失败导致启动崩溃（详见 [04-执行步骤.md](04-执行步骤.md) 阶段2踩坑记录）。`imageData` 因为是可选类型（`Data?`），不需要默认值也能安全迁移。
- 图片存储：直接把 PNG 二进制存进 SwiftData（`imageData: Data?`），没有单独存文件到磁盘再存路径——笔记配图通常不大，这样更简单，数据也跟着笔记一起被 SwiftData 管理（删笔记就自动没有孤儿图片文件）。

### 6. 列表排序与筛选

- 主窗口用 `@Query(sort: \Note.createdAt, order: .reverse)` 自动按时间倒序（已完成）。
- 顶部标签筛选条（`TagFilterBar`），选中的标签用它自己的自定义颜色高亮，"全部"固定用粉色；点击只看某个标签，或点"全部"恢复显示所有笔记（已完成）。
- 标签管理（`TagManagementView`，弹窗形式）：列表里每个标签前有 `ColorPicker` 可以直接改颜色，右边有删除按钮；底部有输入框+颜色选择器+"添加"按钮可以新增标签。删除标签时，用到该标签的笔记会自动变成"未分类"（SwiftData 关系默认 `.nullify` 删除规则）。

### 7. 图片支持（已完成）

- 悬浮框的图片+文字输入区是同一个圆角容器（不是两个分开的框），拖图片进去用 `.onDrop(of: [.image], ...)`，通过 `NSItemProvider.loadObject(ofClass: NSImage.self)` 读取，转成 PNG `Data` 存到 `attachedImageData`。
- 列表、编辑窗口都用 `ThumbnailImageView` 显示缩略图，点一下弹出 `.sheet` 展示大图。
- 曾经做过一个"点按钮直接读剪贴板贴图"的功能，这个按钮无论怎么调整（换图标、加布局保护）都点不动，原因没查清楚，最后直接去掉了，只保留拖拽（够用）。

### 8. 视觉设计（已完成）

详见 [03-设计规范.md](03-设计规范.md)。主题色用 `Theme.swift` 里的 `Color.brandPink` 常量（而不是 Assets.xcassets 的 AccentColor），字体统一 `.system(.body, design: .rounded)`，笔记列表用 `ScrollView + LazyVStack` 卡片样式（`cornerRadius(12)` + 轻阴影）代替普通 `List`。

### 9. 界面语言（已完成）

App 界面文字（按钮、菜单、窗口标题、占位提示）全部是英文，硬编码在代码里，没有做多语言 `Localizable.strings` 系统——这是个人单人使用的项目，不需要支持多语言切换，直接写英文字符串更简单。项目文档（`docs/`、`dev-log/`）保持中文，那是内部工作记录，跟 App 界面本身是两回事。

## 环境要求

- macOS 14 及以上（用户当前 macOS 26.5.1/26.6，Apple Silicon，完全满足）
- 完整版 Xcode（已安装：Xcode 26.6，命令行工具已指向完整版）

## 开发环境注意事项

- 日常调试建议打包成 `.app` 再用 `open` 启动，而不是直接 `swift run`：`swift build` 后把 `.build/arm64-apple-macosx/debug/NoteWidget` 拷贝到 `NoteWidget.app/Contents/MacOS/NoteWidget`，再 `open NoteWidget.app`。`NoteWidget.app/Contents/Info.plist` 已经写好，不需要每次重建。
- 每次改完代码重新运行前，先 `pkill -9 -f NoteWidget` 确认没有残留的旧进程（`ps aux | grep -i notewidget`），否则新旧窗口同时存在会导致测试反馈混乱、难以排查问题。
