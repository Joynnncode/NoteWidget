# 设计规范

对应需求见 [01-需求文档.md](01-需求文档.md)。

## 设计原则

简洁、直观。用户是非技术背景，所有操作要一眼看懂：保存按钮要明显、打开列表的入口要明显、标签选择要明显，不用隐藏手势或猜谜式图标。

## 主题色

- 主色调：**粉色**（`Assets.xcassets/AccentColor`，参考色值 `#FF2D78` 或系统 `.pink` 邻近色），设置为全局 `.tint()`/`.accentColor()`，按钮、选中态、图标默认继承。
- 选中的标签筛选按钮：粉色填充；未选中：描边/灰色。

## 字体

- 系统字体 + 圆角风格（`Font.system(.body, design: .rounded)`），更亲和、不生硬。
- 标题类文字（"保存"按钮、窗口标题）用 `.headline` / `.title3`，其余正文用默认 `.body`。

## 间距与卡片样式

- 8pt 网格：8 / 16 / 24 的内边距递增。
- 笔记卡片：圆角 `cornerRadius(12)` + 轻微阴影 `shadow(radius: 2)`，让悬浮框看起来像一张"便利贴"，笔记列表里每条也保持卡片感。

## 关键界面元素

- **悬浮便签框（FloatingWidgetView）**：
  - 一个文字输入区（`TextEditor`）
  - 一个标签选择器（`Menu`/`Picker`，显示当前标签 + 下拉箭头，不是隐藏手势）
  - 一个明显的粉色"保存"按钮（实心填充，不是纯图标）
  - 一个小的"打开列表"入口（图标按钮，粉色）

- **笔记列表主窗口（NoteListView）**：
  - 顶部：标签筛选条（横向排列的胶囊按钮，含"全部"）
  - 列表：每条笔记显示文字摘录（2-3 行截断）、标签徽标（小圆点或彩色胶囊）、相对时间（如"3 小时前"）
  - 按时间倒序排列，最新在最上面

## 验收标准

- 打开 App 后，不需要任何说明，用户应该能立刻看懂：往哪里输入文字、点哪里保存、点哪里能看到所有笔记、怎么选标签。
- 整体视觉风格统一，粉色贯穿悬浮框和列表窗口，不出现突兀的默认蓝色系统控件。
