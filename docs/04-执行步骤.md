### 踩坑记录（重要，后续开发要注意）
这一阶段调试花了很长时间，记录下来避免以后重复踩坑：

1. **`TextEditor` 在这台机器的系统/工具链组合下有渲染裁切 bug**（文字只显示下半部分）。解决办法：改用 `TextField("...", text:, axis: .vertical).textFieldStyle(.plain).lineLimit(n...m)` 代替 `TextEditor`，视觉效果类似但不裁切。以后新增输入框都用这个写法，不要用 `TextEditor`。
2. **`NSApp.delegate` 读出来是 `nil`**，即使 `@NSApplicationDelegateAdaptor` 已经正常初始化。不要依赖 `NSApp.delegate as? AppDelegate` 来获取 AppDelegate 实例，改用我们自己维护的 `AppDelegate.shared`（在 `applicationDidFinishLaunching` 里赋值的静态属性）。
3. **多个窗口同时用 `NSWindow.level` 置顶时，层级必须匹配**，否则层级更高的窗口会一直挡住层级更低的新窗口，即使新窗口其实已经正常打开了（表现为"点了没反应"）。悬浮框和列表窗口目前都设成 `.floating`。
4. **调试时一定要用 `ps aux | grep NoteWidget` 确认只有一个进程在跑**，再让人测试。`swift run`/直接跑二进制在改代码重新运行前，一定要先杀掉旧进程（`pkill -9 -f NoteWidget`），否则新旧两个窗口同时存在会导致反馈互相矛盾、很难排查。
5. 日常测试建议**打包成 `.app` 再用 `open` 启动**（而不是直接跑 `swift run`/裸执行文件），行为更接近真实使用场景。打包方式：`swift build` 后把 `.build/arm64-apple-macosx/debug/NoteWidget` 拷贝到 `NoteWidget.app/Contents/MacOS/NoteWidget`，`Info.plist` 已经放在 `NoteWidget.app/Contents/` 下了，之后 `open NoteWidget.app` 即可。

### 踩坑记录（补充）
- **给 SwiftData 模型加新字段时，要在属性声明处直接给默认值**（如 `var colorHex: String = "#FF6B9D"`），不要只在 `init` 参数里给默认值——否则旧数据在自动迁移时会因为"缺少必填字段的值"而迁移失败，导致 App 启动直接崩溃报 `Fatal error: 无法创建本地数据库`。这次因为只是测试数据所以直接删了本地数据库重建（`~/Library/Application Support/default.store*`），但以后有真实数据后要避免这种情况，加字段前想清楚要不要给默认值。

- [x] 悬浮窗口加上 `.nonactivatingPanel` 样式和完整 `collectionBehavior`（`.canJoinAllSpaces` + `.fullScreenAuxiliary` + `.stationary` + `.ignoresCycle`）

### 踩坑记录（阶段4）
这一步同样反复调试了很久，记录下来：

1. **只给悬浮框加 `.nonactivatingPanel` + `.fullScreenAuxiliary` 是不够的**——悬浮框在全屏视频上能正常显示了，但点"列表"打开的列表窗口一开始死活不出现在全屏画面里（只能回到普通桌面才看得到）。原因是列表窗口当时还是"会抢焦点"的普通窗口/面板（`.titled` 但没有 `.nonactivatingPanel`）。**列表窗口也要用 `NSPanel` + `.nonactivatingPanel`（可以同时保留 `.titled`/`.closable` 等样式，两者不冲突），行为才会跟悬浮框一致**。
2. **不要在"显示/切换窗口"这类逻辑里调用 `NSApp.activate(ignoringOtherApps: true)`**——如果在全屏视频播放时调用它，会把系统从全屏 Space 切回普通桌面（这也是当时"必须回桌面才能看到列表"的另一个原因）。这个调用只应该在 App **刚启动**时用一次，之后显示/隐藏窗口只用 `makeKeyAndOrderFront` / `orderOut` 就够了。
3. **要在全屏之前就把窗口建好**——如果等到别的 App 已经进入全屏之后才第一次创建新窗口，系统不一定会把这个新窗口纳入那个全屏 Space。列表窗口现在改成跟悬浮框一样，在 App 启动时就建好（先用 `orderOut` 隐藏），点"列表"按钮时只是让已存在的窗口 `makeKeyAndOrderFront`，不再是"点了才临时创建"。
4. 排查这类问题时同样要严格遵守阶段1的经验：每次改一个变量、重新打包、确认只有一个进程再测。
